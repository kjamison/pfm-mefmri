ME-fMRI Pipeline Software Dependencies
=====================================

Purpose
-------

This file records the software stack currently expected and tested by this
repository. The previous version of this document listed tool names only; this
version adds concrete version information so users can reproduce a working
environment on other machines.

Version policy
--------------

- "Pinned/tested" means the version was found directly in this repo, in the
  configured `mefmri_env` conda environment, in the runtime container files, or
  from the current development machine.
- "Not pinned by repo" means the pipeline can call the software, but this repo
  does not currently enforce an exact version for it.
- If you want the closest match to the current tested setup, prefer the
  container image in `docker/Dockerfile.runtime` plus the tedana environment
  listed below.

- `NSI_USE_EXTERNAL_CLI=1`

Only needed if:
- `PFM_STRATEGY=infomap`
- and `PFM_INFOMAP_USE_MATLAB=0`

Required command:
- `infomap`

If infomap is not on PATH, set:
- `PFM_INFOMAP_BINARY=/full/path/to/infomap`

Version status:
- not pinned by repo

D. MATLAB
Only needed if:
- `PFM_STRATEGY=infomap`
- and `PFM_INFOMAP_USE_MATLAB=1`

Version status:
- not pinned by repo


6) Manual Version / Binary Checks
---------------------------------

Core command presence check:

for c in \
  bash awk sed \
  fslmaths fslmerge flirt topup mcflirt fslnvols fslsplit fslroi fslval \
  convert_xfm convertwarp applywarp invwarp slicetimer bet \
  mri_binarize mri_convert mri_surf2surf bbregister tkregister2 \
  wb_command parallel python3 tedana ica_reclassify; do
  command -v "$c" >/dev/null 2>&1 || echo "MISSING: $c"
done

Recommended tedana env check:

conda run -n mefmri_env python -c "import sys, tedana, numpy, nibabel, pandas, scipy, matplotlib; \
print('python==' + sys.version.split()[0]); \
print('tedana==' + tedana.__version__); \
print('numpy==' + numpy.__version__); \
print('nibabel==' + nibabel.__version__); \
print('pandas==' + pandas.__version__); \
print('scipy==' + scipy.__version__); \
print('matplotlib==' + matplotlib.__version__)"

Common version probes:

bash --version | head -n 1
awk -W version | head -n 1
sed --version | head -n 1
cat "${FSLDIR}/etc/fslversion"
recon-all -version
wb_command -version | head -n 6
git -C HCPpipelines-master rev-parse --short HEAD

Conditional checks:

command -v conda >/dev/null 2>&1 || echo "MISSING: conda"
command -v N4BiasFieldCorrection >/dev/null 2>&1 || echo "MISSING: N4BiasFieldCorrection"
command -v infomap >/dev/null 2>&1 || echo "MISSING: infomap"


7) Remaining Gaps
-----------------

These dependencies are still not fully version-pinned by the repo itself:
- ANTs
- SimNIBS / CHARM
- Infomap
- MATLAB
- Python packages installed in `docker/Dockerfile.runtime`

If you want installs on other machines to be substantially easier, the next
practical step is:
- create an explicit environment file for `mefmri_env` (for example
  `environment.yml` or a version-pinned `requirements.txt`)
- pin the pip installs in `docker/Dockerfile.runtime`
- optionally document tested ANTs / SimNIBS / Infomap versions in `README.md`
